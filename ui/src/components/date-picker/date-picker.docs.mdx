# DatePicker Component Documentation

## Overview

A React DatePicker component that provides **complete antd DatePicker behavior** with Dayjs integration. This component offers sophisticated date handling, validation, and user experience patterns.

## When To Use

By clicking the input box, you can select a date from a popup calendar.

## Features

### ✅ Core Features

- **Dayjs Integration**: Full Dayjs support for date manipulation
- **Antd Behavior Parity**: Exact matching of antd DatePicker UX patterns
- **Type Safety**: Full TypeScript support with generic types
- **Real-time Sync**: Input and calendar stay synchronized
- **Validation**: Comprehensive date validation and error handling
- **Accessibility**: Keyboard navigation and screen reader support

### ✅ User Experience

- **Input Typing**: Real-time calendar updates without triggering onChange
- **Calendar Selection**: Immediate date selection with onChange
- **Keyboard Navigation**: Complete keyboard support (Enter, Escape, Tab)
- **Blur Handling**: Smart validation and reversion logic
- **Visual Feedback**: Clear visual states and transitions

## API Reference

### Props

```typescript
interface DatePickerProps<DateValueType extends Dayjs = Dayjs> {
  // Value management
  value?: DateValueType;
  defaultValue?: DateValueType;
  onChange?: (
    date: DateValueType | null | undefined,
    dateString: string,
  ) => void;

  // Display
  format?: string;
  placeholder?: string;
  showTime?: boolean;

  // Modes
  picker?: 'date' | 'year';

  // Calendar UI
  captionLayout?: 'label' | 'dropdown' | 'dropdown-months' | 'dropdown-years';

  // Range
  minDate?: Dayjs;
  maxDate?: Dayjs;

  // Custom behaviors
  /**
   * When a year is selected in overlay mode (picker="date"), commit that year on popover close
   * without requiring the user to pick a day. The committed value follows the current sticky preview.
   */
  commitYearOnClose?: boolean;

  // Validation
  disabledDate?: (date: DateValueType, info: { type: string }) => boolean;

  // State
  disabled?: boolean;
  allowClear?: boolean;
  variant?: string;
  size?: string;
  status?: string;

  // Styling
  className?: string;
  classNames?: Record<string, string>;
  styles?: Record<string, React.CSSProperties>;
}
```

### Behavior Matrix

| Action                 | Input Valid                           | Input Invalid                         | Empty Input                           |
| ---------------------- | ------------------------------------- | ------------------------------------- | ------------------------------------- |
| **Enter**              | ✅ onChange(parsed)                   | ❌ no action                          | ✅ onChange(undefined)                |
| **Escape**             | ✅ onChange(parsed)                   | ❌ just close                         | ❌ just close                         |
| **Blur**               | ✅ onChange(parsed)                   | ✅ revert value                       | ✅ onChange(undefined)                |
| **Calendar Select**    | ✅ onChange(selected)                 | ✅ onChange(selected)                 | ✅ onChange(selected)                 |
| **Calendar Click Fix** | ✅ Prevent blur from closing calendar | ✅ Prevent blur from closing calendar | ✅ Prevent blur from closing calendar |

### Year Selection Behavior

- __picker="year"__
  - Selecting a year commits immediately and closes the popover.
  - Input updates to the start of the selected year.
- __picker="date"__ (overlay year selection)
  - Selecting a year navigates the calendar to that year and applies a sticky preview to the input.
  - The input text is dimmed while previewing (popover open) if preview differs from the committed value at the day level.
  - Commit occurs when:
    - A day is selected, or
    - The popover closes and `commitYearOnClose` is true and a year change is pending.
  - When `captionLayout="dropdown"`, changing the year via dropdown also marks a pending year selection, enabling commit-on-close if configured.

## Usage Examples

### Basic Usage

Basic use case. Users can select or input a date in a panel.

```typescript
import { DatePicker } from '@acme/ui';
import dayjs from 'dayjs';

function App() {
  const [date, setDate] = useState<dayjs.Dayjs | null>(null);

  return (
    <>
    <DatePicker
      value={date}
      onChange={(date) => setDate(date)}
      placeholder="Select date"
    />
    <DatePicker onChange={onChange} picker="year" />
    </>
  );
}
```

### Commit Year On Close (overlay)

Commit the selected year when closing the popover without choosing a day.

```tsx
import { DatePicker } from '@acme/ui';
import dayjs from 'dayjs';

function Example() {
  const [date, setDate] = useState<dayjs.Dayjs | null>(null);
  return (
    <DatePicker
      picker="date"
      value={date}
      onChange={(d) => setDate(d)}
      format="YYYY-MM-DD"
      placeholder="Select date"
      minDate={dayjs().subtract(3, 'year').startOf('year')}
      maxDate={dayjs().add(3, 'year').endOf('year')}
      captionLayout="dropdown"
      commitYearOnClose
    />
  );
}
```

## Implementation Details

### State Management

```typescript
// Main state variables
const [value, setValue] = useState<DateValueType | null>(null);
const [inputValue, setInputValue] = useState<string>("");
const [typedDate, setTypedDate] = useState<Date | undefined>(undefined);
const [month, setMonth] = useState<Date | undefined>(undefined);
```

### Value Synchronization

The component uses `useMergedState` for controlled/uncontrolled mode handling:

```typescript
const [value, setValue] = useMergedState(defaultValue, {
  value: valueProp,
  onChange: (value) => {
    onChange?.(value, value ? value.format(format) : "");
  },
});
```

Input value is synchronized using `useEffect`:

```typescript
useEffect(() => {
  const newInputValue = value ? value.format(format) : "";
  setInputValue(newInputValue);
}, [value, format, setInputValue]);
```

This approach avoids the performance issues of using `value` prop in `useMergedState` while ensuring proper synchronization when external values change.

### Validation Logic

```typescript
// Input validation
const validateInput = (input: string) => {
  const parsed = dayjs(input, format);
  return parsed.isValid() ? parsed : null;
};

// Date validation
const isValidDate = (date: dayjs.Dayjs) => {
  return date.isValid();
};
```

### Event Flow

1. **Input Typing** → Update calendar display only
2. **Enter/Escape** → Validate and trigger onChange
3. **Blur** → Validate and trigger onChange or revert
4. **Calendar Selection** → Always trigger onChange

## Troubleshooting

### Common Issues

1. **Calendar not updating**: Ensure `format` prop matches input format
2. **Validation not working**: Check `disabledDate` function implementation
3. **Type errors**: Ensure proper generic type usage
4. **Calendar click not working**: Blur handling now prevents calendar from closing when clicking calendar days
5. **Blur event issues**: Enhanced blur handling with focus target detection
