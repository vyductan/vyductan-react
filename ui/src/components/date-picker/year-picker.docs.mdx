## Introduction

The YearPicker component allows users to select a year from a decade-based panel. It provides an intuitive interface for year selection with decade navigation, similar to Ant Design's year picker.

## Import

```tsx
import { YearPicker } from "@acme/ui/components/date-picker";
```

## Basic Usage

```tsx
import React, { useState } from "react";
import { YearPicker } from "@acme/ui/components/date-picker";

function BasicYearPicker() {
  const [year, setYear] = useState<number>();

  return (
    <YearPicker
      value={year}
      onChange={setYear}
      placeholder="Select year"
    />
  );
}
```

## API Reference

### YearPicker Props

| Property | Description | Type | Default |
|----------|-------------|------|---------|
| `value` | Current selected year | `number \| Dayjs` | - |
| `onChange` | Callback when year is changed | `(value?: number \| Dayjs) => void` | - |
| `placeholder` | Placeholder text | `string` | `"Select year"` |
| `disabled` | Whether the picker is disabled | `boolean` | `false` |
| `allowClear` | Whether to show clear button | `boolean` | `true` |
| `format` | Format for display | `string` | `"YYYY"` |
| `className` | Additional CSS class | `string` | - |
| `open` | Controlled open state | `boolean` | - |
| `onOpenChange` | Callback when open state changes | `(open: boolean) => void` | - |

## Features

### Decade Navigation

The year picker provides decade navigation with previous/next decade buttons:

- **Previous Decade**: Navigate to the previous 10-year range
- **Next Decade**: Navigate to the next 10-year range
- **Current Year Highlight**: The current year is visually highlighted

### Year Selection

- **12 Years Display**: Shows 12 years at a time (previous decade end + current decade + next decade start)
- **Visual States**: Selected, current year, and decade boundaries are visually distinct
- **Keyboard Navigation**: Full keyboard support for accessibility

### Responsive Design

The year picker adapts to different screen sizes and provides optimal user experience on both desktop and mobile devices.

## Usage Patterns

### Controlled Component

```tsx
const [year, setYear] = useState<number>();

<YearPicker
  value={year}
  onChange={setYear}
/>
```

### Uncontrolled Component

```tsx
<YearPicker
  defaultValue={2024}
  onChange={(year) => console.log(year)}
/>
```

### With Form Integration

```tsx
import { useForm } from "react-hook-form";

function YearForm() {
  const { control } = useForm();

  return (
    <Controller
      name="birthYear"
      control={control}
      render={({ field }) => (
        <YearPicker
          {...field}
          placeholder="Select birth year"
        />
      )}
    />
  );
}
```

## Accessibility

- **Keyboard Navigation**: Full keyboard support with arrow keys, Enter, and Escape
- **Screen Reader Support**: Proper ARIA labels and descriptions
- **Focus Management**: Logical tab order and focus trapping

## Migration from Ant Design

If you're migrating from Ant Design's DatePicker.YearPicker, here are the key differences:

| Ant Design | @acme/ui |
|------------|----------|
| `picker="year"` | Built-in year picker |
| `format="YYYY"` | `format="YYYY"` (default) |
| `allowClear` | `allowClear={true}` (default) |
| `disabled` | `disabled={false}` (default) |

### Migration Example

**Ant Design:**
```tsx
<DatePicker
  picker="year"
  value={value}
  onChange={onChange}
  placeholder="Select year"
/>
```

**@acme/ui:**
```tsx
<YearPicker
  value={value}
  onChange={onChange}
  placeholder="Select year"
/>
```
