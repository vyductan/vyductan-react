# Button

A versatile button component with multiple variants, sizes, and styles for all your interaction needs.

## Features

- 🎨 **Multiple Variants** - Solid, outlined, dashed, filled, text, and link styles
- 🎯 **Color Options** - Primary, danger, and custom color support
- 📏 **Flexible Sizes** - Small, default, large, and icon button sizes
- ⚡ **Loading State** - Built-in loading indicator support
- 🔗 **Link Support** - Use as a button or link with `href` prop
- ♿ **Accessible** - Full keyboard navigation and ARIA support
- 🎭 **Polymorphic** - Use `asChild` to render as any element

## API Reference

### Button Props

| Property    | Type                                                                 | Default     | Description                                                    |
| ----------- | -------------------------------------------------------------------- | ----------- | -------------------------------------------------------------- |
| `type`      | `"default" \| "primary" \| "dashed" \| "link" \| "text"`             | `"default"` | Button type (maps to color/variant combination)                |
| `color`     | `"default" \| "primary" \| "danger" \| "pink" \| "purple" \| "cyan"` | -           | Button color                                                   |
| `variant`   | `"solid" \| "outlined" \| "dashed" \| "filled" \| "text" \| "link"`  | -           | Button variant style                                           |
| `size`      | `"sm" \| "default" \| "lg" \| "icon" \| "icon-sm" \| "icon-lg"`      | `"default"` | Button size                                                    |
| `danger`    | `boolean`                                                            | `false`     | Whether the button represents a dangerous action               |
| `loading`   | `boolean`                                                            | `false`     | Whether the button is in loading state                         |
| `disabled`  | `boolean`                                                            | `false`     | Whether the button is disabled                                 |
| `htmlType`  | `"button" \| "submit" \| "reset"`                                    | `"button"`  | The native HTML button type                                    |
| `href`      | `string`                                                             | -           | If provided, button will render as a link                      |
| `asChild`   | `boolean`                                                            | `false`     | Render button as child component (for polymorphic composition) |
| `icon`      | `React.ReactNode`                                                    | -           | Icon to display in the button                                  |
| `shape`     | `"default" \| "circle" \| "round"`                                   | `"default"` | Button shape                                                   |
| `className` | `string`                                                             | -           | Additional CSS classes                                         |

## Migration from Ant Design

```tsx
// Ant Design
import { Button } from 'antd';
<Button type="primary">Primary</Button>
<Button type="primary" danger>Danger</Button>
<Button type="primary" loading>Loading</Button>

// @acme/ui Button
import { Button } from '@/components/ui';
<Button type="primary">Primary</Button>
<Button type="primary" danger>Danger</Button>
<Button type="primary" loading>Loading</Button>
```

### Key Differences

- Uses **Tailwind CSS** for styling instead of Less
- Supports **polymorphic composition** with `asChild` prop
- **Color and variant** props can be used independently or together
- **Icon buttons** require `aria-label` for accessibility
- **Loading state** uses built-in spinner component
